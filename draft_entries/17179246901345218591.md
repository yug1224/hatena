「開発環境現状確認」という記事が流行っているようなので、流れに乗ってみる。

## エディタ / コーディングエージェント

[https://cursor.com/ja:embed:cite]

エディタはCursorを使っている。VS Code + Clineで頑張っていた時期もあったが、Cursorの方がコンテキスト管理やインデックス機能が充実していて、開発体験が良かったので移行した。

常用しているモデルはOpus 4.5。他のエディタとコーディングエージェントの組み合わせも試してみたが、Cursor + Opus 4.5の組み合わせが今のところ自分の中での最適解になっている。

開発に必要なリポジトリはまとめて、マルチルートワークスペースで運用しているのがポイント。Cursorが複数リポジトリの情報をまとめてインデックスしてくれるし、1つのウィンドウの中でリポジトリを跨いで開発を進めてくれるので重宝している。フロントエンドとバックエンドが別リポジトリの場合でも、両方の文脈を理解した上で実装してくれるのがありがたい。

### MCP

MCPも積極的に活用している。

[https://github.com/upstash/context7:embed:cite]

[https://github.com/github/github-mcp-server:embed:cite]

[https://developers.notion.com/docs/mcp:embed:cite]

[https://help.figma.com/hc/ja/articles/32132100833559-Figma-MCP%E3%82%B5%E3%83%BC%E3%83%90%E3%83%BC%E3%81%AE%E3%82%AC%E3%82%A4%E3%83%89:embed:cite]

[https://storybook.js.org/addons/@storybook/addon-mcp:embed:cite]

MCPの中でも特に使っているのがContext7とGitHub MCP。Context7は公式ドキュメントを読み込ませて、ライブラリの使い方を指示するのに使っている。GitHub MCPはPRの説明文を書かせたり、CI失敗の対応をさせたり。

ドキュメントやチケット管理はNotionにまとめているので、Notion MCPも必須。UIの実装ではFigma MCPとStorybook MCPを組み合わせて使うこともある。

## その他AIツール

コーディングエージェント以外にも、いくつかのAIツールを使い分けている。

GeminiとNotebook LMは、気になる記事や資料を突っ込んで、サマリを作ってもらってから読み込んだりしている。長文を効率的にインプットするのに便利。URLを貼っても扱えないコンテンツがある場合は、Raycast Browser Extensionでページの情報を取得してコンテキストに渡すようにしている。

[https://www.raycast.com/browser-extension:embed:cite]

Nano Banana Proは、ADRやPRの概要をインフォグラフィック・グラフィックレコーディングさせるのに使っている。視覚的にまとまった画像が欲しいときに重宝する。

## ターミナル / シェル

ターミナルは、mac標準のターミナル + zshが基本。

ターミナルアプリはいろいろあるけれど、結局乗り換えずに標準を使い続けている。GhosttyやWezTermも話題だし、試してはみたものの、標準ターミナルで特に困っていないので一軍にはならなかった。

zshはpreztoで管理していて、zsh-autosuggestionsとzsh-completionsを入れている。プロンプトはシンプルに `kylewest`テーマを使っている。Starshipも気になるが、preztoで十分間に合っているので手を出せていない。

よく使うエイリアスとしては、`ls`を`eza`に、`cat`を`bat`に置き換えている。色付きで見やすいし、gitステータスも表示できるので便利。

## パッケージマネージャー / dotfiles

パッケージマネージャーは Homebrew がメインだが、mise も併用している。

[https://github.com/jdx/mise:embed:cite]

miseで入れられるものは、だんだんとHomebrewから移しても良いかもなと思っている。Node.jsやDenoのバージョン管理はmiseに任せている。プロジェクトごとに `.mise.toml` でバージョンを固定できるのが便利。

### dotfiles

[https://github.com/yug1224/dotfiles:embed:cite]

dotfilesは一応管理している。そんなに変わっていないけど、最新状態はコミットできていない。

dotfiles managerとしてchezmoiなども気になるが、結局シンプルにGNU Stowを使っている。複雑なことはしていないので、これで十分かなと。

## マウス / その他周辺機器

### トラックボール

[https://www.elecom.co.jp/products/M-XT3URBK.html:embed:cite]

[https://amzn.asia/d/bfgki2x:embed:cite]

トラックボールも壊れる度にリピートしているエレコムのM-XT3URBK。ボールはペリックスのボールに交換していて、標準のボールよりも滑りが良いので気に入っている。マウスだと手首を動かす必要があるが、トラックボールなら親指だけで操作できるので、長時間作業でも疲れにくい。

### マイク

[https://row.hyperx.com/ja/products/hyperx-flipcast-streaming-xlr-usb-dynamic-microphone:embed:cite]

最近までHyperX QuadCast Sを使っていたが、壊れたのでHyperX FlipCast を導入。XLR対応だけどUSB接続でも使えるので、オーディオインターフェースがなくても手軽に使える。

### ヘッドホン

[https://www.soundhouse.co.jp/products/detail/item/25126/:embed:cite]

ヘッドホンは、サウンドハウスのCPH7000。軽いので長時間付けていても疲れないし、コスパも良い。

## まとめ

AI周りの環境が大きく変わった一方で、ターミナルやキーボードなどの基盤となる部分は変わっていない。mac標準ターミナル + zsh + REALFORCEという組み合わせは、もう10年以上使い続けている気がする。
