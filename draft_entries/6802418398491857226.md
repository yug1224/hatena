2025年も残りわずかとなった。この記事では、私にとって大きな転換点となった一年を振り返ってみたいと思う。

EMからICへ戻ろうか悩んでいる方、スタートアップへの転職を考えている方、ブランクを経て開発現場に復帰しようとしている方。そんな方々の参考になれば嬉しい。もちろん、これはあくまで私自身の経験に基づいた振り返りであり、同じ選択がすべての人に適しているとは限らない。

## 2025年を一言で表すと

**「変容」** の一年だった。

5年以上続けてきたエンジニアリングマネージャー（EM）からプロダクトエンジニア（IC）へ。大規模な組織から創業間もないスタートアップへ。そして、AIを相棒として開発を進めるスタイルへ。

単なる「転職」や「復帰」ではなく、エンジニアとしての自分自身が変わった感覚がある。RPGで例えるなら、同じ職業に戻る「ジョブチェンジ」ではなく、これまでの経験を統合した「クラスチェンジ」に近いかもしれない。

## キャリアの転換点：EMからICへ

### なぜ転職を決意したのか

2025年7月、[Dress Code株式会社](https://www.dress-code.com/ja)にジョインした。

もともと「いつかはアーリーのスタートアップに挑戦したい」という思いはあった。しかし、5年以上も開発の現場から離れていた自分が、今さらICとして通用するのか。正直なところ、その夢は半ば諦めかけていた。

そんな中、AIの登場とVibe Codingの流行があった。「AIの伴走があれば、ブランクがある自分でもまたICに戻れるのでは」——そう考えたのが転職を決意したきっかけだった。

[https://zenn.dev/dress_code/articles/be0aaf9be396a4:embed:cite]

### 不安と期待が入り混じったスタート

入社直後、正直に言うと「しがみついている」状態だった。

チームメンバーが1つのPRを出す間に、自分は何も出せていない。全盛期の3分の1程度の速度しか出せない悔しさ。「本当にやっていけるのか」という不安でいっぱいだった。

でも、諦めるわけにはいかなかった。AIの力を借りながら、なんとか食らいついていく日々が始まった。

## 変容の軌跡：半年間での成長

### 数字で振り返る

| YYYY/MM  | Pull Request |
| -------- | -----------: |
| 2025/07  |          6件 |
| 2025/08  |         29件 |
| 2025/09  |         51件 |
| 2025/10  |         50件 |
| 2025/11  |         37件 |
| 2025/12  |         47件 |
| **合計** |    **220件** |

7月の6件から始まり、半年で220件のPRをマージできるようになった。チームでは年間3,900件以上のPRがマージされる環境。この圧倒的なスピード感の中で振り落とされず、ついていけるようになったのは大きな成長だった。

### アンラーニングとリスキリング

半年間で、自分のスキルセットは大きく変わった。

**手放したもの**：特定の技術スタック（フロントエンドなど）へのこだわり。Dress Codeではフルスタック・フルサイクルなプロダクト開発が求められる。最初は戸惑ったが、この環境に身を置くことで、自然とこだわりは薄れていった。

**獲得したもの**：AIマネジメントスキル。これが一番大きな変化だった。AIを活用した開発は、単に「AIにコードを書かせる」ことではない。プランニングから始めて、タスクを細かく分解して指示し、進捗を確認しながら進める。これはプロジェクトマネジメントに近い感覚で、EMやPjM時代の経験が思わぬ形で活きている。

[https://zenn.dev/dress_code/articles/eb8d9afcd86391:embed:cite]

### やりきったと感じた瞬間

10月にリリースした「ネットワーク管理/Networks」のプロダクト開発を担当したとき、ようやく「人並みに成果を出せるようになった」と感じた。

フロントエンドからバックエンドまで一気通貫でプロダクトリリースをやりきれたこと。チームのスピードに追いつけたこと。他のメンバーのフォローなく完遂できたこと。今見ると改善したいところはたくさんあるが、あの瞬間が自信につながった。

## 技術面のハイライト

### 開発環境の爆速化

oxlint、oxfmt、tsgoを導入して、開発環境が劇的に改善した。

| 項目                      | Before | After | 変化     |
| ------------------------- | ------ | ----- | -------- |
| Lint（バックエンド）      | 52秒   | 4秒   | **-92%** |
| Typecheck（バックエンド） | 105秒  | 36秒  | **-65%** |

初めてCIのログを見たときは二度見した。「え、もう終わったの？」という感覚。コーヒーを淹れて戻ってきてもまだ終わっていない、そんな待ち時間から解放された。

[https://zenn.dev/dress_code/articles/d655cd7a43b936:embed:cite]

### AI活用の日常化

開発ではCursor/Claudeを、プライベートではGeminiを活用している。設計から実装まで一貫したワークフローを実現できるようになった。

ドメインモデリングではPlantUMLをAIと対話しながら作成し、そこからPrismaのスキーマを生成。APIの実装は要件と参考実装を伝えて進め、UIの実装にはFigma MCPを活用。AIが「できる後輩」のような存在になっている。

## アウトプット活動の充実

### アドベントカレンダー参加

Dress Code Advent Calendar 2025に複数記事を投稿し、チームで25日間完走できた。また、エンジニア転職Advent Calendar 2025にも参加した。

[https://zenn.dev/dress_code/articles/0f38e10e85fb6b:embed:cite]

### hatenaブログ再開

2月からhatenaブログも再開した。QiitaやZennとは違い、プログラミング以外のことも書いていこうと思っている。チームビルディング、リーダーシップ、マネジメント、キャリア、日々の生活のこと。ざっくばらんに何でも書ければいいかなと。

### 技術広報・イベント参加

入社直後からSRE NEXT 2025のスポンサーブース運営に参加した。他にもFindy感謝祭、Vue Fes、gunmaweb、Japan IT Weekなど、さまざまなイベントに足を運んだ。

社外のエンジニアと交流すると、普段知り得ないノウハウを学べたり、さまざまな刺激をもらえる。これは転職前から大切にしていたことで、これからも続けていきたい。

## プライベート：仕事と生活のバランス

### 子ども部屋のDIY

今年一番嬉しかった瞬間は、子どもたちと一緒に間仕切り壁をDIYしたときかもしれない。

GeminiとNano Banana Proで設計図を作り、ホームセンターで材料を調達。中学生の長男が予想以上に戦力で、対等に意見を出し合いながらの作業は心強く、頼もしかった。

完成したとき、子どもたちの「すごい！壁ができた！」という声。仕事でもプライベートでもAIを活用して、家族との時間も大切にできた一年だった。

[https://zenn.dev/dress_code/articles/b085b7826f1c56:embed:cite]

### 趣味の充実

競馬のG1観戦、アニメ（ルリドラゴン、ジョジョ、MFゴースト）、ウイスキー。仕事だけでなく、趣味の時間も充実していた。

仕事と生活のバランスを保ちながら、リフレッシュする時間を持てたのは良かったと思う。

## おわりに：2026年への展望

この半年は、自信と技術力を取り戻す期間だった。

2026年は、これまでの経験と、取り戻した技術力・開発力を掛け合わせて成果を出していきたい。「ここは任せろ！」と自他ともに認める領域を確立すること。プロダクト開発で価値貢献しつつ、ピープルマネジメントをかけ合わせた領域で力を発揮していくこと。

もしEMからICへの転身を考えていたり、ブランクを経て開発現場に戻ろうとしている方がいれば、伝えたいことがある。必ずしも昔と同じに戻る必要はない。AIという強力なパートナーと共に、新しいエンジニアとして「変容」する道もある。

2025年、変容の一年をありがとう。2026年も、ごきげんにいこう。
